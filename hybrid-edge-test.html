hybrid-edge-test

Confirms that the hybrid engine (Web Audio on top of silent HTMLAudioElement) survives stop/restart and silent-track looping.
Test A

Start → Stop → Start (fresh speaker route)

Plays the metronome for 3 seconds (Web Audio clicks on top of silent backing track), stops everything, waits 2 seconds, then starts again. If you hear clicks in both the first AND second play phase, the speaker route re-establishes cleanly. If the second phase is silent, we need to keep the backing track running even when "stopped."
—
Test B

Long loop — Web Audio stability across silent-track boundaries

Plays continuously for 30 seconds. Silent backing track is only 3 seconds long, so it loops roughly 10 times during the test. Web Audio fires a steady 120 BPM click the entire time. Listen for any glitch in the click timing when the silent track loops — the backing track's loop gap shouldn't matter (our audio is synthesized independently), but worth confirming. The readout below tracks every loop boundary the browser reports.

Test C

Background-playing backing track (always-on variant)

Different strategy: start the silent backing track once, never stop it. When user taps "Play Clicks," we only start/stop the Web Audio clicks. The silent track runs continuously regardless of metronome state. This should be the most bulletproof approach if Test A fails.
